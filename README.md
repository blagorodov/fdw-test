# Фордевинд — тестовое задание

## Цель

Разработать веб-приложение на Laravel для голосования за фотографии аукционных автомобилей.

## Функционал

### Страница 1 — Голосование

- Выпадающий список всех доступных моделей.
- После выбора модели отображаются две разные случайные фотографии этой модели (фотографии не должны повторяться в цикле).
- Пользователь выбирает: «Левая нравится больше» / «Правая нравится больше».
- После голосования сразу показывается новая случайная пара (рекомендуется без полной перезагрузки страницы).
- Дополнительно (по желанию), при использовании jQuery: зум фото через jQuery ezPlus методом Tints.

### Страница 2 — Статистика

- Фильтры: модель автомобиля и диапазон годов (от — до).
- Вывод:
  - таблица с карточками автомобилей (фото и данные), количеством голосов за каждую;
  - общая сумма голосов по всем автомобилям выбранной модели в указанном диапазоне лет.

## Запуск окружения (Docker)

Для запуска проекта необходимо:
1. Скачать репозиторий в отдельный каталог на сервере.
2. Запустить команду `make init`.
3. Для того, чтобы избежать CORS ошибки при загрузке изображений, отредактируйте на сервере файл .env - в APP_URL вместо localhost необходимо вставить реальный IP или домен сервера. 
4. Открыть http://...IP-сервера.../

Для управления окружением нужна команда `make`. Убедитесь, что утилита установлена.

**Установка `make`, если ее у вас нет:**

| Платформа | Как поставить                                                                                                                     |
|-----------|-----------------------------------------------------------------------------------------------------------------------------------|
| **macOS** | Обычно уже есть вместе с инструментами разработчика. Если команды `make` нет: [Homebrew](https://brew.sh/) — `brew install make`. |
| **Linux (Debian/Ubuntu и производные)** | `sudo apt update && sudo apt install -y make`.                                                                                    |
| **Windows** | Удобнее всего в **WSL2** (Ubuntu): как в строке Linux выше.                                                                       |

## Команды make

| Команда      | Действие                                                                                                                               |
|-------------|----------------------------------------------------------------------------------------------------------------------------------------|
| `make init` | Поднять контейнеры как `make up`, затем выполнить инициализацию проекта (копирует .env, загружает vendor, запускает миграции и сидер). |
| `make up`   | Поднять контейнеры в фоне (`docker compose up -d`), проект Compose: `fdw`.                                                             |
| `make down` | Остановить контейнеры и убрать их (`docker compose down`).                                                                             |
| `make remove` | Остановить контейнеры, удалить их и **все тома** проекта (`docker compose down -v`), в том числе данные PostgreSQL.                    |
| `make build` | Пересобрать все контейнеры с `build` в `docker-compose.yml`, если менялись Dockerfile или контекст сборки.                      |
